最好的种树时间是十年前，其次是现在。写博客也是。

## 1. 前言 (Introduction)


### 1.1 为什么要写博客？——建立你的“数字花园”

在碎片化信息爆炸的时代，为什么我们依然需要一个独立博客？

- **建立个人技术品牌：** 博客是程序员最好的“动态简历”。它不仅展示了你的技术栈，更记录了你思考问题的逻辑和解决问题的过程，能极大地提升个人影响力。
- **深度学习与知识复盘：** 根据费曼学习法，最好的学习方式是“教会别人”。将碎片化的知识整理成文章，是查漏补缺、内化知识的绝佳途径。
- **沉淀与输出：** 社交媒体的信息是流动的，而博客是静止的。它是你的数字资产，多年后回看，你会清晰地看到自己的成长轨迹。
- **掌握话语权：** 告别第三方平台的审核限制、广告干扰和突如其来的关停风险。在自己的博客里，你拥有绝对的自主权和排版权。

### 1.2 为什么选择 Hexo？——轻量与极致的平衡

在众多的博客框架（如 WordPress, Hugo, Jekyll）中，Hexo 脱颖而出成为很多开发者的首选，原因在于：

- **极速生成：** 基于 Node.js 开发，拥有极快的静态页面生成速度，上百篇文章瞬间渲染完成。
- **Markdown 原生支持：** 专为写作而生。你只需专注于文字，排版交给 Markdown，展示交给 Hexo。
- **生态繁荣：** 拥有数百种精美主题（如 NexT, Fluid, Butterfly）和丰富的插件，无论是添加评论系统、搜索功能还是 SEO 优化，都能一键实现。
- **零成本托管：** 配合 GitHub Pages 或 Vercel，你无需购买服务器，无需维护数据库，真正实现“零元购”搭建全网可访问的站点。
- **纯粹且安全：** 生成的是静态 HTML，没有任何数据库注入风险，访问速度极快。

### 1.3 本文你能学到什么？——从零到一的实战地图

本文将拒绝枯燥的理论，以实战为导向，手把手带你完成以下任务：

1. **环境搭建：** 配置 Node.js 与 Git，打好底层基础。
2. **核心构建：** 从零初始化 Hexo 框架，让博客在你的本地跑起来。
3. **个性化定制：** 挑选并配置一套高颜值主题，让你的博客与众不同。
4. **内容创作：** 掌握 Markdown 写作技巧及 Hexo 常用指令。
5. **全球发布：** 将博客部署至 GitHub Pages，让全世界都能通过专属域名看到你的文字。

## 2. 环境准备 (Prerequisites)

在正式搭建博客之前，我们需要在电脑上安装三件“神兵利器”：**Node.js**（动力源）、**Git**（运输车）以及一个 **GitHub 账号**（停机坪）。

### 2.1 安装 Node.js：Hexo 的心脏

Hexo 是基于 Node.js 运行的，因此这是最基础的环境。

- **下载地址：** 访问 [Node.js 官网](https://nodejs.org/)。
- **版本选择：** 强烈建议选择 **LTS (Long Term Support) 版本**（如 v18 或 v20），因为 LTS 版本更稳定，不容易出现兼容性问题。
- **安装步骤：** 像安装普通软件一样点击“下一步”即可。安装时记得勾选“Add to PATH”选项。
- **验证安装：** 打开终端（Windows 推荐使用 PowerShell 或 CMD，macOS 使用 Terminal），输入以下命令：*如果能看到版本号（如 v18.16.0），说明安装成功。*
    
    ```bash
    node -v
    npm -v
    
    ```
    

> 🚀 进阶技巧： 由于国内访问 npm 官方源较慢，建议安装后立即配置淘宝镜像（或华为云镜像）来加速后续的插件下载：
> 
> 
> `npm config set registry https://registry.npmmirror.com`
> 

### 2.2 安装 Git：代码管理与自动部署

Git 负责将你本地写好的文章“推”到远程服务器上。

- **下载地址：** 访问 [Git 官网](https://git-scm.com/)。
- **安装步骤：** 默认设置即可。安装完成后，在右键菜单中看到 "Git Bash Here" 即表示安装成功。
- **验证安装：** 在终端输入：
    
    ```bash
    git --version
    
    ```
    
- **配置身份（必做）：** 第一次使用 Git 需要告诉它你是谁，否则无法提交代码。输入以下命令：
    
    ```bash
    git config --global user.name "你的用户名"
    git config --global user.email "你的邮箱地址"
    
    ```
    

### 2.3 GitHub 账号与仓库准备：你的免费服务器

我们需要 GitHub 提供的 **GitHub Pages** 服务来托管你的静态博客。

- **注册账号：** 访问 [GitHub官网](https://github.com/) 注册（如果已有账号请忽略）。
- **新建仓库 (New repository)：**
    - **仓库名 (Repository Name) 极其重要：** 必须按照这个格式：`你的用户名.github.io`。
    - *例如：* 如果你的用户名是 `tech-sharen`，那么仓库名必须是 `tech-sharen.github.io`。
    - **权限选择：** 必须设为 **Public**（公开）。
    - **其他勾选：** 无需勾选 README 或 License，直接点击 `Create repository`。
- **SSH 公钥配置（进阶推荐）：** 为了让你每次部署博客时不需要输入账号密码，建议配置 SSH Key。这能极大提升后续的使用体验。

## 3. Hexo 安装与初始化 (Getting Started)

环境配好后，只需几条简单的指令，就能让你的博客在本地跑起来。

### 3.1 安装 Hexo-cli 工具

`hexo-cli` 是 Hexo 的命令行工具，安装了它，你才能在终端里使用 `hexo` 开头的命令。

在终端执行：

```bash
npm install -g hexo-cli

```

- **注意：** `g` 代表全局安装。安装完成后，你可以输入 `hexo -v` 查看版本，确认是否安装成功。

### 3.2 初始化博客文件夹

找一个你觉得舒服的地方（比如 `D:\Blog` 或 `~/Documents/Blog`），在该目录下打开终端，执行以下操作：

```bash
# 初始化一个名为 my-blog 的文件夹（名字可以自定义）
hexo init my-blog

# 进入这个文件夹
cd my-blog

# 安装必要的依赖包
npm install

```

- **发生了什么？** 执行 `hexo init` 时，Hexo 会自动从 GitHub 拉取一套完整的“样板房”文件到你的文件夹里。

### 3.3 本地预览：见证奇迹的时刻

现在，你的博客其实已经“搭建”好了。执行以下命令开启预览：

```bash
hexo s

```

- `s` 是 `server` 的缩写。
- **查看结果：** 打开浏览器，访问 `http://localhost:4000`。
- 你现在应该能看到一个简洁大气的默认页面（Landscaping 主题），上面有一篇名为 "Hello World" 的示例文章。

> 💡 小贴士： 在终端按 Ctrl + C 可以停止本地服务器。
> 

### 3.4 目录结构详解：别被这些文件吓到

初始化完成后，你会看到文件夹里多了一堆文件。了解它们的用途，能让你后续定制时事半功倍：

- **`_config.yml` (网站配置文件)：** 博客的“大脑”。你的网站标题、作者信息、URL 格式、推送设置都在这里修改。
- **`package.json`：** 记录了 Hexo 版本以及你安装了哪些插件。
- **`scaffolds` (脚手架)：** 文章的模板。比如你新建一篇文章，它会自动包含日期、标签等基础信息。
- **`source` (资源文件夹)：** **这是你以后待得最多的地方。**
    - `_posts` 文件夹：存放你写的所有 Markdown 文章。
- **`themes` (主题文件夹)：** 存放各种皮肤。如果你下载了新主题，就放在这里。
- **`public` (公共文件夹)：** 静态生成的产物。执行 `hexo g` 后，Hexo 会把你的 Markdown 转换成 HTML 存放在这里。**注意：不要手动修改这个文件夹里的内容。**

---

**常用命令速查表：**

- `hexo clean`：清除缓存（如果你发现修改没生效，执行它）。
- `hexo g` (generate)：生成静态文件。
- `hexo s` (server)：本地预览。
- `hexo d` (deploy)：部署上线。

## 4. 博客基础配置 (Configuration)

在 Hexo 根目录下，有一个至关重要的文件：`_config.yml`。我们称之为**站点配置文件**。几乎所有全局性的设置（网站名、作者、SEO 设定）都在这里完成。

> ⚠️ 重要提醒： YAML 格式对空格极其敏感。在修改时，请务必保证每一个冒号 : 后面都有一个英文空格，否则会报错。
> 

### 4.1 打造你的个人名片：站点信息修改

找到文件开头的 `Site` 部分，根据你的情况进行修改：

```yaml
# Site - 网站基础信息
title: 某某的技术博客           # 网站标题
subtitle: 每一个不曾起舞的日子，都是对生命的辜负 # 副标题/座右铭
description: 专注于后端开发、架构设计与好物分享  # 网站描述（对 SEO 搜索排名很重要）
keywords: Java, SpringCloud, Hexo, 技术博客 # 网站关键词
author: 你的笔名               # 作者名字
language: zh-CN               # 网站语言，设置为中文
timezone: Asia/Shanghai       # 时区，建议设为上海

```

- **小贴士：** `language` 的设置直接决定了主题中“阅读更多”、“评论”等文字的显示语言。

### 4.2 设置站点 URL：让搜索引擎找到你

找到 `URL` 部分。如果你打算部署在 GitHub Pages，请务必修改这里：

```yaml
# URL
url: https://你的用户名.github.io  # 以后你的线上博客地址
root: /                          # 根路径，一般不用动

```

### 4.3 优化永久链接 (Permalinks)：让 URL 更优雅

Hexo 默认生成的文章链接格式是 `年/月/日/文章标题/`。

- **默认格式的问题：** 如果你以后修改了文章的发布日期，链接就会变，导致别人之前收藏的链接失效；且链接层级太深，不利于搜索引擎抓取。
- **优化建议：** 建议修改为 `:title.html`。

```yaml
# Directory
permalink: posts/:title.html    # 优化后的链接：域名/posts/文章名.html

```

- **这样做的好处：** 无论你以后如何调整分类或日期，文章的 URL 都将保持唯一且简洁，极大地提升了 **SEO（搜索引擎优化）** 表现。

### 4.4 开启代码高亮

为了让技术文章里的代码块更好看，检查 `highlight` 部分（Hexo 7.0 以后默认推荐使用 `shiki` 或 `highlight.js`）：

```yaml
syntax_highlighter: highlight.js
highlight:
  line_number: true    # 显示行号
  enable: true

```

---

如果说前面的步骤是搭建“毛坯房”，那么**主题美化**就是博客的“软装”阶段。Hexo 最迷人的地方就在于它极其丰富的主题生态，只需几行配置，你的博客就能从“简陋”变得“惊艳”。

## 5. 主题美化：给博客“换装” (Themes)

Hexo 默认的 `landscape` 主题虽然稳重，但对于追求个性的开发者来说显然不够。换主题是每个 Hexo 玩家的必经之路。

### 5.1 挑选心仪的主题

你可以从以下渠道寻找灵感：

- **官方主题库：** [Hexo Themes](https://hexo.io/themes/)（收录了上百款主题）。
- **高人气推荐：**
    - **Fluid：** 现代化的 Material Design 风格，配置极简，文档齐全（**推荐新手首选**）。
    - **Butterfly：** 功能极其强大，特效华丽，支持高度定制。
    - **NexT：** Hexo 界长盛不衰的经典，极简主义，适合硬核技术博客。

### 5.2 更换主题的通用流程

更换主题通常有两种方式：`git clone`（传统方式）和 `npm install`（现代方式，方便升级）。以 **Fluid** 为例：

1. **下载主题：**
    
    在博客根目录下执行：
    
    ```bash
    npm install hexo-theme-fluid --save
    
    ```
    
    *（如果是传统方式，则是将代码 clone 到 `themes/` 文件夹下）*
    
2. **激活主题：**
    
    打开**根目录**下的 `_config.yml`（站点配置文件），找到 `theme` 字段：
    
    ```yaml
    # 将 landscape 修改为 fluid
    theme: fluid
    
    ```
    
3. **创建主题配置文件（推荐）：**
    
    为了方便升级，建议在根目录新建一个 `_config.fluid.yml` 文件。Hexo 会自动用这个文件的内容覆盖主题内部的默认配置，这样你就不需要修改 `node_modules` 或 `themes` 里的源代码了。
    

### 5.3 个性化微调：细节决定质感

现在，打开你的主题配置文件（例如 `_config.fluid.yml`），开始打磨细节：

- **配置导航栏 (Menu)：**让读者能快速找到分类和关于页。
    
    ```yaml
    menu:
      Home: /
      Archives: /archives/
      Categories: /categories/
      About: /about/
    
    ```
    
- **更换 Favicon & Logo：**将你的头像或网站图标放入 `source/img/` 目录下，并在配置文件中指定路径。
- **社交链接：**配置你的 GitHub、知乎、BiliBili 等链接，增加互动。
- **全局字体与配色：**许多现代主题支持一键切换“暗黑模式”或修改主色调。

### 5.4 验证效果

每次修改配置后，记得执行以下命令组合：

```bash
hexo clean && hexo s

```

- **为什么要 clean？** Hexo 会缓存之前的静态页面，修改主题配置后如果不 clean，浏览器里看到的可能还是旧样子。

> “如果博客图片较多，建议使用图床（如 Github + PicGo）来提升加载速度。”
> 

博客的排版、配置固然重要，但**内容才是核心**。Hexo 采用 Markdown 作为写作语言，这对于程序员来说几乎零学习成本，且排版极其优雅。

## 6. 开启创作：写下第一篇文章 (Writing)

万事俱备，现在让我们写下第一篇博文。

### 6.1 新建一篇文章

在博客根目录下打开终端，输入：

```bash
hexo new "我的第一篇技术文章"

```

- **发生了什么：** Hexo 会在 `source/_posts/` 目录下自动生成一个文件：`我的第一篇技术文章.md`。
- **提示：** 如果文件名包含空格，请务必使用英文双引号括起来。

### 6.2 认识 Front-matter：文章的元数据

打开刚刚生成的 `.md` 文件，你会看到顶部有一段由 `---` 包围的代码，这就是 **Front-matter**。它告诉 Hexo 如何处理这篇文章。

```yaml
---
title: 我的第一篇技术文章
date: 2023-10-27 10:00:00
tags: [Hexo, 教程]
categories: 技术心得
description: 这是我使用 Hexo 搭建博客的第一篇记录，分享一下心得。
---

```

- **title：** 文章标题（可以和文件名不同）。
- **date：** 发布日期（通常自动生成）。
- **tags：** 标签（一篇文章可以有多个标签）。
- **categories：** 分类（建议每篇文章只有一个主分类）。
- **top_img / index_img：** 很多主题支持在这里设置文章的封面图。

### 6.3 Markdown 写作建议

你现在可以使用任何编辑器（推荐 **Typora** 或 **VS Code**）开始书写正文。

对于技术博客，以下三个 Markdown 语法最常用：

1. **分级标题：** 使用 `#`、`##` 来构建目录。
2. **代码块：** 使用 ` ` ` 语法。Hexo 会根据你在 `_config.yml` 的设置自动高亮代码。
3. **引用：** 使用 `>` 来标注参考资料或重点提示。

### 6.4 插入图片的两种“姿势”

在静态博客中处理图片是一个难点，这里推荐两种方案：

**方案 A：本地资源文件夹（适合新手）**

1. 修改 `_config.yml` 中的 `post_asset_folder: true`。
2. 以后执行 `hexo new` 时，会额外生成一个同名的文件夹。
3. 将图片放入该文件夹，然后在 Markdown 中引用：`![描述](图片文件名.jpg)`。

**方案 B：图床方案（进阶推荐，加载飞快）**

将图片上传到专门的存储服务器（如 阿里云 OSS、腾讯云 COS，或者通过 **PicGo + GitHub** 搭建免费图床）。

- **优点：** 减少博客源码体积，提升全球访问速度。
- **引用方式：** `![描述](https://你的图床链接.jpg)`。

### 6.5 预览与发布

每写完一段，都可以执行 `hexo s` 并在浏览器刷新查看效果。Hexo 支持“热更新”，你保存 Markdown 文件后，本地预览通常会自动刷新。

---

1. **“阅读更多”截断：**
    
    如果不希望首页显示文章全文，可以在正文中插入 `<!-- more -->`。
    
    - `<!-- more -->` 之前的内容会作为摘要显示在首页。
    - 之后的文字点击“阅读更多”后才能看到。
2. **Emoji 表情支持：**
    
    技术文档太枯燥？尝试加入一些 Emoji 🚀（Win 键 + `.` 即可唤出），会让页面生动很多。
    
3. **文章命名建议：**
    
    虽然 `hexo new` 支持中文名，但建议文件名使用 **英文或拼音**（例如 `hello-world.md`），而把真正的中文标题写在 Front-matter 的 `title` 里。这能避免在某些系统下出现 URL 乱码或路径解析错误。

## 7. 部署上线 (Deployment)

Hexo 最强大的功能之一就是**一键部署**。通过简单的配置，你可以将生成的静态页面瞬间发布到 GitHub Pages。

### 7.1 准备“停机坪”：GitHub 仓库

在环境准备阶段，我们已经提到了仓库创建，这里再强调两个核心细节：

- **仓库命名：** 必须是 `你的用户名.github.io`。
- **仓库属性：** 必须设为 **Public**（公开），否则 GitHub Pages 无法免费提供静态托管服务。

### 7.2 安装部署插件

Hexo 本身并不直接具备“上传到 Git”的功能，需要安装一个官方支持的插件：

在博客根目录下执行：

```bash
npm install hexo-deployer-git --save

```

### 7.3 配置站点部署信息

打开**根目录**下的 `_config.yml`（站点配置文件），滑到文件最底部，找到 `deploy` 字段，修改如下：

```yaml
# Deployment
deploy:
  type: git
  repo: git@github.com:你的用户名/你的用户名.github.io.git
  branch: main

```

- **repo 的选择：** 建议使用 **SSH 地址**（如上所示），这样配合 SSH Key 可以实现免密推送。如果你还没配置 SSH，也可以使用 HTTPS 地址。
- **branch：** 目前 GitHub 默认主分支名为 `main`，请确保此处与你仓库的分支名一致。

### 7.4 一键发布：见证奇迹的时刻

现在，只需要在终端输入这一行“神咒”：

```bash
hexo g -d

```

- **`g` (generate)：** 生成静态文件。
- **`d` (deploy)：** 部署到远程仓库。
- **执行过程：** Hexo 会自动扫描你的文章，转换成 HTML，然后通过 Git 推送到你的 GitHub 仓库。

> 检查结果： 稍等 1-2 分钟（GitHub 部署需要时间），访问 https://你的用户名.github.io，你应该就能看到你的博客已经在公网上运行了！
> 

### 7.5 进阶：绑定个人域名 (Optional)

如果你觉得 `.github.io` 后缀不够专业，想拥有类似 `www.yourname.com` 的域名：

1. **购买域名：** 在阿里云、腾讯云或 GoDaddy 购买。
2. **设置 DNS 解析：** 在域名后台添加一条 `CNAME` 记录，指向 `你的用户名.github.io`。
3. **Hexo 配置：** 在 `source` 目录下新建一个名为 **`CNAME`**（无后缀）的文件，内容只写你的域名（如 `www.yourname.com`）。
    - *为什么要放在 source 下？* 因为每次 `hexo clean` 都会清空 public，只有放在 source 下，部署时才会自动同步到 GitHub 根目录，保证域名设置不丢失。
4. **GitHub 设置：** 在仓库的 `Settings -> Pages` 菜单中，勾选 **Enforce HTTPS**，让你的博客更安全。

## 8. 源码备份：别让你的心血付诸东流

在 Hexo 的世界里，有一个最基础但也最致命的误解：**以为执行了 `hexo d`，你的博客就安全了。**

### 8.1 警示：`hexo d` 推送的到底是什么？

当你执行 `hexo d` 时，Hexo 只是把 `public` 文件夹（即生成的 HTML 静态页面）推送到 GitHub。

- **它不包括：** 你的 `_config.yml` 配置文件、你的主题源码、以及最重要的——你辛苦撰写的 Markdown 原稿。
- **后果：** 一旦你的电脑硬盘损坏或重装系统，你将彻底失去对博客的控制权。你看着线上的博客，却再也无法修改它。

### 8.2 最佳实践：双分支备份法

我们建议利用同一个 GitHub 仓库，通过不同的“分支”来分别存放**静态网页**和**博客源码**。

1. **准备 `.gitignore`：**
    
    在博客根目录创建一个 `.gitignore` 文件（通常 Hexo 自带了），确保以下文件**不会**被备份（因为它们可以通过命令自动生成）：
    
    ```
    node_modules/
    public/
    .deploy_git/
    db.json
    
    ```
    
2. **创建并推送源码分支：**
    
    在你的本地博客根目录打开终端，执行以下命令：
    
    ```bash
    git init
    git remote add origin git@github.com:你的用户名/你的用户名.github.io.git
    
    # 创建一个名为 hexo 的新分支用于存放源码
    git checkout -b hexo
    
    # 提交并推送
    git add .
    git commit -m "Backup hexo source code"
    git push origin hexo
    
    ```
    
3. **现在的仓库结构：**
    - **`main` 分支：** 存放生成的静态 HTML（由 `hexo d` 自动更新）。
    - **`hexo` 分支：** 存放你的整个 Hexo 源码项目（由你手动 `git push` 更新）。

### 8.3 多端协作：换台电脑也能写

如果你买了新电脑，或者想在公司摸鱼写博客，只需几步即可找回环境：

1. **克隆源码分支：**
    
    ```bash
    git clone -b hexo git@github.com:你的用户名/你的用户名.github.io.git
    cd 你的用户名.github.io
    
    ```
    
2. **恢复环境：**由于我们备份了 `package.json`，你不需要重新一个个安装插件，只需一行命令：
    
    ```bash
    npm install
    # 如果你全局没装 hexo-cli，也记得装一下
    npm install -g hexo-cli
    
    ```
    
3. **开始写作：**现在的环境和老电脑上一模一样，你可以继续 `hexo new`、`hexo g -d` 了。

## 9. 结语 (Conclusion)

恭喜你！如果你完整地走到了这一步，你已经成功在浩瀚的互联网世界中圈下了一块属于自己的“数字领地”。从环境配置到源码备份，你不仅掌握了一套高效的工具流，更完成了一次从“信息消费者”向“内容生产者”的角色转变。

### 9.1 搭建只是开始，内容才是灵魂

在博客圈有一个常见的陷阱：**“折腾不止，写作不始”**。很多开发者会花费大量的时间在更换主题、调色、折腾特效上，却迟迟不写第一篇文章。

请记住：

- **博客的价值在于文字：** 一个只有精美外壳而没有思想内涵的博客，就像一座空房子。
- **保持输出：** 写博客不是为了向世界展示你有多聪明，而是为了整理自己的思绪。哪怕只是记录一个困扰你一小时的 Bug 解决方法，对未来的你和社区的后来者都有巨大的意义。
- **不积跬步无以至千里：** 不需要追求篇篇都是深度长文。持续、真实地记录你的技术成长，时间会赋予你的博客惊人的复利效应。

### 9.2 开启你的技术社交之旅

独立博客虽然是一座“孤岛”，但它通过链接和搜索与整个互联网相连。

- **保持好奇心：** 随着你写的文章越来越多，你会发现自己对技术的理解变得更加通透。
- **积极互动：** 在你的博文中引用他人的观点，在评论区回复读者的疑问。你会发现，博客是你建立技术圈影响力的敲门砖。
